# MCP 활용 가이드 및 호출 규칙

## 🎯 MCP 서버별 활용 전략

### 1. **playwright** - 웹 자동화
**호출 키워드**: `웹 테스트`, `브라우저 자동화`, `페이지 스크린샷`, `웹 스크래핑`
```markdown
- 웹사이트 테스트 자동화가 필요할 때
- 브라우저에서 특정 동작을 시뮬레이션할 때
- 웹 페이지의 스크린샷이나 스냅샷이 필요할 때
- 웹 스크래핑이나 데이터 수집이 필요할 때
```

### 2. **memory** - 지식 그래프 관리
**호출 키워드**: `메모리 저장`, `지식 그래프`, `정보 기억`, `컨텍스트 관리`
```markdown
- 중요한 정보를 기억해야 할 때
- 프로젝트 관련 지식을 체계적으로 관리할 때
- 이전 대화 내용을 참조해야 할 때
- 사용자 선호도나 설정을 저장할 때
```

### 3. **supabase** - 데이터베이스 관리
**호출 키워드**: `데이터베이스`, `Supabase`, `SQL 쿼리`, `데이터 관리`
```markdown
- 데이터베이스 스키마를 확인하거나 수정할 때
- SQL 쿼리를 실행해야 할 때
- Supabase 프로젝트를 관리할 때
- 데이터 마이그레이션이 필요할 때
```

### 4. **chrome-devtools** - 개발자 도구
**호출 키워드**: `개발자 도구`, `성능 분석`, `디버깅`, `네트워크 모니터링`
```markdown
- 웹 페이지의 성능을 분석할 때
- 네트워크 요청을 모니터링할 때
- JavaScript 오류를 디버깅할 때
- 페이지 로딩 시간을 측정할 때
```

### 5. **filesystem** - 파일 시스템
**호출 키워드**: `파일 읽기`, `파일 쓰기`, `디렉토리 구조`, `파일 관리`
```markdown
- 프로젝트 파일을 읽거나 수정할 때
- 디렉토리 구조를 확인할 때
- 파일을 생성하거나 삭제할 때
- 파일 시스템 작업이 필요할 때
```

### 6. **github** - GitHub 연동
**호출 키워드**: `GitHub`, `리포지토리`, `이슈 관리`, `PR 생성`
```markdown
- GitHub 리포지토리를 관리할 때
- 이슈나 PR을 생성/수정할 때
- 코드 리뷰가 필요할 때
- GitHub API를 사용해야 할 때
```

### 7. **sequential-thinking-tools** - 순차적 사고
**호출 키워드**: `단계별 분석`, `논리적 사고`, `문제 해결`, `계획 수립`
```markdown
- 복잡한 문제를 단계별로 분석할 때
- 논리적 사고 과정이 필요할 때
- 체계적인 문제 해결이 필요할 때
- 프로젝트 계획을 수립할 때
```

### 8. **task-master-ai** - 작업 관리
**호출 키워드**: `작업 관리`, `할 일 목록`, `프로젝트 관리`, `일정 관리`
```markdown
- 프로젝트 작업을 체계적으로 관리할 때
- 할 일 목록을 만들고 추적할 때
- 작업 우선순위를 정할 때
- 프로젝트 일정을 관리할 때
```

### 9. **figma-api** / **Figma** / **figma-desktop** - 디자인 도구
**호출 키워드**: `Figma`, `디자인`, `UI/UX`, `프로토타입`, `디자인 시스템`
```markdown
- UI/UX 디자인 작업이 필요할 때
- Figma 파일을 관리할 때
- 디자인 시스템을 구축할 때
- 프로토타입을 만들 때
```

## 🚀 MCP 호출 패턴

### 기본 호출 형식
```
[키워드] + [구체적 요청사항] + [MCP 서버명]
```

### 예시 호출 문구
1. **웹 테스트**: "웹사이트 로그인 기능을 playwright로 테스트해줘"
2. **메모리 저장**: "이 프로젝트 설정을 memory에 저장해줘"
3. **데이터베이스**: "사용자 테이블 구조를 supabase로 확인해줘"
4. **성능 분석**: "현재 페이지의 성능을 chrome-devtools로 분석해줘"
5. **파일 관리**: "프로젝트 구조를 filesystem으로 확인해줘"
6. **GitHub 관리**: "새로운 이슈를 github로 생성해줘"
7. **단계별 분석**: "이 문제를 sequential-thinking-tools로 단계별로 분석해줘"
8. **작업 관리**: "프로젝트 작업을 task-master-ai로 관리해줘"
9. **디자인**: "UI 컴포넌트를 figma로 디자인해줘"

## 📋 MCP 활용 체크리스트

### 작업 시작 전
- [ ] 어떤 MCP 서버가 필요한지 확인
- [ ] 적절한 키워드로 호출
- [ ] 필요한 권한이나 설정이 있는지 확인

### 작업 중
- [ ] MCP 서버의 응답을 적절히 활용
- [ ] 오류 발생 시 대안 MCP 서버 고려
- [ ] 결과를 다른 MCP 서버와 연동하여 활용

### 작업 완료 후
- [ ] 중요한 정보는 memory에 저장
- [ ] 작업 결과를 적절히 문서화
- [ ] 다음 작업을 위한 준비

## ⚠️ 주의사항

1. **개인키 보안**: supabase, github, task-master-ai 등은 개인키가 필요하므로 보안에 주의
2. **파일 시스템 권한**: filesystem은 지정된 디렉토리만 접근 가능
3. **로컬 서버**: figma-desktop은 로컬 서버가 실행 중이어야 함
4. **네트워크 연결**: 일부 MCP 서버는 인터넷 연결이 필요

## 🔄 MCP 서버 간 연동 예시

1. **웹 개발 워크플로우**:
   ```
   filesystem → playwright → chrome-devtools → memory
   (코드 확인 → 테스트 → 성능 분석 → 결과 저장)
   ```

2. **디자인 개발 워크플로우**:
   ```
   figma → filesystem → github → memory
   (디자인 → 코드 구현 → 버전 관리 → 지식 저장)
   ```

3. **데이터 관리 워크플로우**:
   ```
   supabase → sequential-thinking-tools → task-master-ai → memory
   (데이터 분석 → 논리적 사고 → 작업 계획 → 지식 저장)
   ```

이 가이드를 참고하여 MCP 서버들을 효과적으로 활용하세요!
