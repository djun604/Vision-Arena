# MCP 호출 규칙 및 패턴

## 🎯 MCP 서버별 호출 키워드 매핑

### 웹 자동화 및 테스트
- **playwright**: `웹테스트`, `브라우저`, `자동화`, `스크린샷`, `스크래핑`
- **chrome-devtools**: `성능분석`, `디버깅`, `네트워크`, `개발자도구`

### 데이터 및 저장소
- **memory**: `기억`, `저장`, `지식그래프`, `컨텍스트`, `메모리`
- **supabase**: `데이터베이스`, `SQL`, `DB`, `테이블`, `쿼리`
- **filesystem**: `파일`, `폴더`, `디렉토리`, `읽기`, `쓰기`

### 개발 및 협업
- **github**: `리포지토리`, `이슈`, `PR`, `커밋`, `브랜치`
- **sequential-thinking-tools**: `분석`, `단계별`, `논리적`, `사고`, `계획`
- **task-master-ai**: `작업관리`, `할일`, `프로젝트`, `일정`, `태스크`

### 디자인 및 UI/UX
- **figma-api**: `디자인`, `UI`, `UX`, `프로토타입`, `컴포넌트`
- **Figma**: `피그마`, `디자인시스템`, `와이어프레임`
- **figma-desktop**: `데스크톱`, `로컬피그마`

## 📝 호출 문구 패턴

### 1. 직접 호출 패턴
```
"[키워드]로 [작업내용]해줘"
예: "playwright로 웹사이트 테스트해줘"
```

### 2. 조건부 호출 패턴
```
"[상황]이니까 [MCP서버]를 사용해서 [작업]해줘"
예: "성능 문제가 있으니까 chrome-devtools로 분석해줘"
```

### 3. 연동 호출 패턴
```
"[MCP1]로 [작업1]하고 [MCP2]로 [작업2]해줘"
예: "filesystem으로 파일 읽고 memory에 저장해줘"
```

### 4. 문제 해결 호출 패턴
```
"[문제]를 해결하기 위해 [MCP서버]를 활용해줘"
예: "웹페이지 오류를 해결하기 위해 chrome-devtools를 활용해줘"
```

## 🔄 워크플로우별 MCP 조합

### 웹 개발 워크플로우
1. **코드 확인**: `filesystem`으로 파일 읽기
2. **테스트 실행**: `playwright`로 자동화 테스트
3. **성능 분석**: `chrome-devtools`로 성능 측정
4. **결과 저장**: `memory`에 테스트 결과 저장

### 데이터 분석 워크플로우
1. **데이터 조회**: `supabase`로 데이터베이스 쿼리
2. **논리적 분석**: `sequential-thinking-tools`로 단계별 분석
3. **작업 계획**: `task-master-ai`로 작업 관리
4. **지식 저장**: `memory`에 분석 결과 저장

### 디자인 개발 워크플로우
1. **디자인 확인**: `figma`로 디자인 파일 확인
2. **코드 구현**: `filesystem`으로 파일 수정
3. **버전 관리**: `github`로 커밋 및 푸시
4. **문서화**: `memory`에 개발 과정 저장

## ⚡ 빠른 호출 명령어

### 자주 사용하는 조합
- `웹테스트`: playwright + chrome-devtools
- `데이터분석`: supabase + sequential-thinking-tools
- `파일관리`: filesystem + memory
- `프로젝트관리`: task-master-ai + github
- `디자인개발`: figma + filesystem

### 긴급 상황 대응
- **오류 발생**: `chrome-devtools`로 디버깅
- **성능 문제**: `playwright` + `chrome-devtools`로 분석
- **데이터 문제**: `supabase`로 데이터베이스 확인
- **파일 문제**: `filesystem`으로 파일 상태 확인
